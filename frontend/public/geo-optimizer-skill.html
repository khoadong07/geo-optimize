geo-optimizer-skill

An open-source toolkit that audits and optimizes websites for citation by AI answer engines — ChatGPT, Perplexity, Gemini, Google AI Overviews, and Claude.

What it does

The core command, geo audit, fetches a live URL and scores it 0-100 on how ready it is to be crawled and cited by AI answer engines — the same kind of technical readiness that used to matter for search-engine crawlers, adapted for LLM-based ones. It ships as a CLI, a Python library, a GitHub Action, an MCP server, and an Astro framework integration.

The 8 core scoring categories used by GeoBase

CategoryWhat it checks
Robots.txtWhether known AI crawlers (GPTBot, ClaudeBot, PerplexityBot, Google-Extended, etc.) are explicitly allowed.
llms.txtPresence of the emerging /llms.txt convention for structuring a site for AI indexing.
Schema JSON-LDStructured data — Website/WebApp/FAQ schema types found on the page.
Meta tagsTitle and description tags AI systems use to understand page intent.
ContentHeading structure (H1 presence, hierarchy) and general content quality signals.
Page signalsLanguage declaration and other page-level technical signals.
AI discoveryWhether AI-facing discovery endpoints are present.
Brand entityWhether the brand name is referenced consistently across the page.

Additional signals available, not yet surfaced

Negative signals

CTA density, popups, thin content, keyword stuffing, boilerplate ratio.

WebMCP readiness

Whether the site exposes agent-callable tools/actions.

CDN / AI-crawler check

Whether a CDN is silently blocking AI crawler user agents.

JS rendering

Whether critical content requires JavaScript to appear (AI crawlers often don't execute JS).

Trust stack

Aggregate trust/authority scoring.

Prompt injection detection

Hidden text or HTML comments attempting to manipulate AI crawlers.

RAG chunk & embedding quality

How well content would chunk and embed for retrieval-augmented generation.

Content decay & platform citation

Whether content is going stale, and citation behavior per platform.

How GeoBase integrates it

GeoBase does not reimplement this tool's logic — it installs the real Python package into the backend image and calls it directly.

  • The backend spawns geo audit --url <domain> --format json as a subprocess, in a background job (a full audit can take 10-60s).
  • The parsed score, 8-category breakdown, and recommendations are stored in MongoDB and shown on the Gap & Citation page.
  • The project's website is set once on Target Position, then audits can be re-run any time.
